## 📂 Project Structure & Mapping

### 1. Directory Blueprint

You should create these directories that save past paper questions:

| Directory | Contained Papers | Generated Output |
| :--- | :--- | :--- |
| **P1** | Pure Mathematics 1 (2015-2022) | P1 chapter files |
| **P3** | Pure Mathematics 3 (2015-2022) | P3 chapter files |
| **M1** | Mechanics (2015-2022) | M1 chapter files |
| **S1** | Probability and Statistics 1 (2015-2022) | S1 chapter files |
| **FP1** | Further Pure Mathematics 1 (2020-2022) | FP1 chapter files |
| **FP2** | Further Pure Mathematics 2 (2020-2022) | FP2 chapter files |
| **FP** | FP1 & FP2 Historical (2015-2019) | Source files only |
| **FM** | Further Mechanics (2020-2022) | FM chapter files |
| **M2** | Further Mechanics Historical (2015-2019) | Source files only |
| **FS** | Further Probability and Statistics (2020-2022)| FS chapter files |
| **FX** | FM & FS Historical (2015-2019) | Source files only |
| **IG0606**| IGCSE Additional Mathematics (2019-2024)| IG0606 chapter files|
| **books** | N/A | Contains all compiled exercise books |

### 2. Core Control Files

* **`Questions by topic.xlsx`**  
  The main database tracking the mapping and sorting of past paper questions into corresponding chapters.
* **`Questions by topic subset.xlsx`**  
  A curated, high-yield subset database. If the full question bank is too heavy for your cohort, use this to freely select *critical core questions*.
* **`questions_by_topic_book.py`**  
  Python script to compile and generate a **comprehensive book-style PDF** containing all sorted questions for a selected subject.
* **`questions_by_topic_chapters.py`**  
  Python script to slice and output **individual chapter PDF sheets**, optimized for daily homework assignments.

## 📖 Operational Guide

### Step 1: Record & Format Question Data

When updating mappings in `Questions by topic.xlsx` or `Questions by topic subset.xlsx`, strictly adhere to the following syntax:
* **Multiple Questions**: Separate independent questions using commas, e.g., `1, 3, 6`.
* **Sub-questions / Parts**: Use a dot separator for specific sub-questions, e.g., `3.b`, `7.2`.
* **Historical Paper Exceptions**: For legacy **FP** and **FX** papers (2015–2019), where the final question often contains dual macro-sections, use suffixes `a` and `b`, e.g., `11a, 11b`.

### Step 2: Configure Script Parameters

Open either `questions_by_topic_book.py` or `questions_by_topic_chapters.py` in your preferred IDE. Locate and modify the parameter block:

```python
# ==========================================
# SYSTEM PARAMETERS CONTROL PANEL
# ==========================================
subject_name = r'S1'
start_year = 2021
end_year = 2015
subset = False
```

Parameter Reference Terminal:
- `subject_name` (String): Specify the target syllabus module:
  - `P1`: Pure Mathematics 1
  - `P3`: Pure Mathematics 3
  - `M1`: Mechanics
  - `S1`: Probability and Statistics 1
  - `FP1`: Further Pure 1
  - `FP2`: Further Pure 2
  - `FM`: Further Mechanics
  - `FS`: Further Probability & Statistics
  - `IG0606`: IGCSE Additional Mathematics 0606
- `start_year` / `end_year` (Integer): Specifies the paper range.
  - Pedagogical Tip: Setting `start_year = 2021` is highly recommended, allowing recent past papers (2022–2024) to be safely reserved for standalone final mock exams.
  - Ensure `end_year` is set no earlier than `2015`.
- `subset` (Boolean):
  - `True`: Compiles data using the full database (`Questions by topic subset.xlsx`).
  - `False`: Compiles data using the curated dataset (`Questions by topic.xlsx`).

### Step 3: Trigger Compilation

Execute the script via your Python IDE, or directly fire up your terminal and run:

```bash
python questions_by_topic_book.py
```

or

```bash
python questions_by_topic_chapters.py
```
